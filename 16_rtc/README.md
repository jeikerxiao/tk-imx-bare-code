# 16_rtc — 实时时钟实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第十六个实验——**实时时钟（RTC）**的完整代码。该实验实现I.MX6ULL内部SNVS（Secure Non-Volatile Storage）RTC模块的驱动，实现年月日时分秒的获取与设置功能。RTC是嵌入式系统中用于记录实时时间的关键外设，即使在系统断电时也能通过备用电池维持计时。

## 3. 关键原理简述

### 3.1 SNVS RTC架构
I.MX6ULL的RTC位于SNVS（Secure Non-Volatile Storage）模块内，特点：
- **独立时钟域**：使用32.768kHz晶振，不受主频影响
- **备用电池供电**：主电源断电时通过备用电池维持计时
- **时间计数器**：内部维护秒计数器，可转换为年月日时分秒

### 3.2 时间读写操作
RTC通过以下寄存器实现时间读写：
- **SNVS_HPCR**：控制寄存器
- **SNVS_HPSR**：状态寄存器
- **SNVS_HPRTCMR** / **SNVS_HPRTCLR**：实时时钟计数器（高32位/低32位）
- **秒计数器**：从1970年1月1日00:00:00开始的秒数

### 3.3 时间格式转换
秒计数器与年月日时分秒的转换：
1. 将秒计数器转换为`struct tm`结构体
2. 考虑闰年、每月天数等规则
3. 支持设置时间和读取时间两种操作

## 5. 下载与验证步骤

1. 编译生成`rtc.bin`
2. 使用`imxdownload`工具烧录至SD卡：
```bash
./imxdownload rtc.bin /dev/sdX
```
3. SD卡插入开发板，设置SD卡启动模式
4. 上电启动

## 6. 预期运行现象

开发板上电后：
- LCD屏幕显示当前时间（年月日时分秒）
- 时间每秒更新
- 可通过按键设置时间

验证标准：时间显示准确，每秒更新，说明RTC驱动正常。

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| 时间不走 | RTC时钟未使能 | 检查SNVS时钟是否已使能 |
| 时间复位 | 备用电池未安装 | 安装CR2032备用电池 |
| 时间不准 | 晶振频率偏差 | 检查32.768kHz晶振是否正常工作 |
| 时间显示错误 | 闰年计算错误 | 检查时间转换函数中的闰年判断逻辑 |
