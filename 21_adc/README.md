# 21_adc — ADC模数转换实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第二十一个实验——**ADC模数转换**的完整代码。
该实验实现I.MX6ULL内部ADC（Analog-to-Digital Converter）模块的驱动，通过ADC采集外部模拟电压并转换为数字量。
ADC是嵌入式系统中将模拟信号转换为数字信号的关键外设，广泛应用于传感器数据采集、电池电压监测等场景。

## 3. 关键原理简述

### 3.1 I.MX6ULL ADC架构
I.MX6ULL内部集成两个12位ADC模块（ADC1和ADC2），特点：
- **分辨率**：12位
- **采样率**：最高1Msps
- **输入通道**：多路模拟输入通道
- **转换模式**：单次转换、连续转换、触发转换

### 3.2 ADC寄存器操作
- **ADC_HC0**：控制寄存器，配置转换通道和模式
- **ADC_HS**：状态寄存器，反映转换状态
- **ADC_R0**：结果寄存器，存储转换后的数字值
- **ADC_CFG**：配置寄存器，设置时钟和分辨率
- **ADC_GC**：通用控制寄存器，使能ADC模块

### 3.3 电压转换公式
```
Voltage = (ADC_Value / 4095) × Vref
```
其中`Vref`为参考电压（通常为3.3V）。

### 3.4 采样与转换流程
1. 初始化ADC时钟和引脚
2. 配置ADC模块（分辨率、时钟、通道）
3. 启动ADC转换
4. 等待转换完成（轮询或中断）
5. 读取结果寄存器，计算电压值
6. 显示或处理数据

## 5. 下载与验证步骤

1. 编译生成`adc.bin`
2. 使用`imxdownload`工具烧录至SD卡：
```bash
./imxdownload adc.bin /dev/sdX
```
3. SD卡插入开发板，设置SD卡启动模式
4. 确保ADC输入引脚已连接待测电压（如电位器）
5. 上电启动

## 6. 预期运行现象

开发板上电后：
- LCD屏幕显示ADC采集的电压值
- 调节电位器时，电压值变化
- 电压值以数字形式显示（如"1.65V"）

验证标准：电压值随输入变化而变化，说明ADC驱动正常。

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| ADC读数全0 | ADC未使能 | 检查ADC_GC寄存器使能位 |
| ADC读数不变 | 输入通道错误 | 确认ADC_HC0中通道选择正确 |
| ADC读数跳变 | 采样时间不足 | 增加采样时间配置 |
| 电压值不准确 | 参考电压偏差 | 测量实际Vref值，校准计算公式 |
| ADC转换不完成 | 时钟未配置 | 检查ADC时钟源和分频配置 |
