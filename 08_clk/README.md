# 08_clk — 系统时钟配置实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第八个实验——**系统时钟配置**的完整代码。
该实验在按键输入实验基础上，新增系统时钟初始化模块，学习如何通过软件将I.MX6ULL的主频从默认的396MHz提升到标准的528MHz。
理解I.MX6ULL复杂的时钟树结构（包括PLL、PFD、分频器等），掌握时钟配置对系统性能和外设工作的影响。

## 3. 关键原理简述

### 3.1 I.MX6ULL时钟树结构
I.MX6ULL的时钟系统由以下核心组件构成：
- **XTAL**：24MHz外部晶振，作为系统时钟源
- **PLL（锁相环）**：
  - `PLL1`：ARM内核主频PLL，输出528MHz
  - `PLL2`（System PLL）：固定528MHz，供给系统总线
  - `PLL3`（USB1 PLL）：固定480MHz，供给USB等外设
  - `PLL5`（Video PLL）：供给显示接口
- **PFD（Phase Fractional Divider）**：将PLL输出分频得到多种时钟频率
- **CCM（Clock Controller Module）**：时钟控制器，管理时钟门控和分频

### 3.2 主频提升流程
1. **切换时钟源**：先将ARM内核时钟源切换至XTAL（24MHz），确保PLL配置期间系统稳定运行
2. **配置PLL1**：设置PLL1输出为528MHz（24MHz × 22，即`DIV_SELECT = 22`）
3. **配置PFD**：设置各PFD分频系数，确保外设时钟正常
4. **切换回PLL1**：将ARM内核时钟源重新切换至PLL1输出
5. **设置分频系数**：配置`CACRR[ARM_PODF]`等分频寄存器，最终得到528MHz内核主频

### 3.3 时钟配置与延时函数的关系
主频从396MHz提升到528MHz后，基于空循环的延时函数周期会发生变化。若主频提升后延时函数未重新校准，实际延时时间会变短。后续实验将使用硬件定时器实现精确延时。

1. 编译生成`clk.bin`
2. 使用`imxdownload`工具烧录至SD卡：
```bash
./imxdownload clk.bin /dev/sdX
```

4. 上电启动

## 6. 预期运行现象

开发板上电后：
- 系统主频从默认396MHz提升至528MHz
- LED0以约500ms为周期闪烁（由于主频提升，实际闪烁频率可能略有变化）
- 按键控制蜂鸣器功能正常工作

验证标准：程序正常运行，LED闪烁和按键响应无明显异常，说明系统时钟配置成功。

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| 程序卡死/不启动 | PLL配置错误导致时钟不稳定 | 检查PLL1的`DIV_SELECT`值是否为22，`CCSR[step]`是否正确切换 |
| 主频未提升 | 时钟源未切换回PLL1 | 确认配置完成后将`CCSR[pll1_sw_clk_sel]`切换回PLL1 |
| 外设异常 | PFD配置错误导致外设时钟异常 | 检查PFD分频系数设置，确保外设时钟在有效范围内 |
| LED闪烁频率异常 | 主频变化后延时函数未校准 | 重新计算空循环次数，或使用硬件定时器实现精确延时 |
