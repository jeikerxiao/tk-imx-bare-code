# 11_key_filter — 定时器按键消抖实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第十一个实验——**定时器实现按键消抖**的完整代码。该实验在EPIT定时器基础上，新增按键消抖（keyfilter）驱动模块，学习如何使用硬件定时器替代软件延时实现按键消抖。相比传统的软件空循环消抖，定时器消抖不占用CPU资源，消抖期间CPU可执行其他任务，是高效嵌入式按键处理的标准实现方法。

## 3. 关键原理简述

### 3.1 按键消抖的必要性
机械按键在按下和释放时，触点会产生持续数毫秒的抖动，导致单次按键被多次检测。若不进行消抖处理，会出现：
- 单次按键触发多次响应
- 按键状态不稳定
- 程序逻辑混乱

### 3.2 软件延时消抖的缺陷
传统的软件消抖通过空循环延时10ms~20ms实现，存在以下问题：
- **CPU占用**：延时期间CPU处于空转状态，浪费计算资源
- **实时性差**：延时期间无法响应其他事件
- **精度低**：受编译优化和主频影响，延时时间不准确

### 3.3 定时器消抖原理
使用EPIT定时器实现按键消抖的流程：
1. **按键中断触发**：按键按下产生外部中断
2. **关闭外部中断**：防止抖动期间重复触发
3. **启动定时器**：设置定时器超时时间为10ms（消抖时间）
4. **定时器中断处理**：10ms后定时器中断触发，此时读取按键状态
5. **状态确认**：若按键仍为按下状态，则判定为有效按键
6. **重新使能外部中断**：等待下一次按键触发

### 3.4 与Linux驱动消抖机制的对比
本实验的定时器消抖机制与Linux内核中的按键驱动消抖原理一致：
- 使用定时器替代忙等待
- 中断+定时器的组合实现可靠消抖
- 消抖期间CPU可处理其他任务

## 6. 预期运行现象

开发板上电后：
- LED0以约500ms为周期闪烁
- 按下KEY0按键时，蜂鸣器状态翻转（通过定时器消抖后响应）

验证标准：按键响应可靠，无重复触发，说明定时器消抖机制工作正常。

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| 按键仍多次触发 | 消抖时间不足 | 增大EPIT定时器超时时间，建议10ms~20ms |
| 按键无响应 | 外部中断未重新使能 | 检查定时器中断处理函数中是否重新使能了按键外部中断 |
| 按键响应延迟 | 消抖时间过长 | 减小定时器超时时间，通常5ms~15ms即可 |
| 定时器中断未触发 | EPIT配置错误 | 参考实验10检查EPIT时钟源、比较值等配置 |
