# 12_highpreci_delay — 高精度延时实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第十二个实验——**高精度延时**的完整代码。
该实验使用GPT（General Purpose Timer）定时器替代软件空循环实现精确延时，最低延时精度可达微秒级。
高精度延时是许多外设驱动（如I2C、SPI、LCD时序控制）的基础功能，对延时精度有严格要求的场景（如通信协议时序）至关重要。

## 3. 关键原理简述

### 3.1 GPT定时器架构
I.MX6ULL内部集成多个GPT定时器，本实验使用GPT1实现高精度延时：
- **32位递增计数器**：自由运行模式，计数值从0递增至`0xFFFFFFFF`
- **时钟源**：可选择IPG时钟（66MHz）或其他时钟
- **捕获/比较功能**：支持输入捕获和输出比较
- **预分频器**：1~4096分频

### 3.2 高精度延时实现
延时函数`delayms()`的实现原理：
1. **初始化**：配置GPT1时钟源和预分频
2. **读取当前计数值**：通过`GPT1_CNT`寄存器获取当前计数值
3. **计算目标计数值**：根据需要的延时时间和时钟频率计算目标计数值
4. **循环等待**：不断读取`GPT1_CNT`，直到计数值达到目标值

计算公式：
```
目标计数值 = 当前计数值 + 延时时间(ms) × 时钟频率(Hz) / 1000
```

### 3.3 与软件延时的性能对比
| 指标 | 软件空循环延时 | GPT硬件延时 |
|------|-------------|------------|
| 最小延时 | 受编译优化影响，不稳定 | 可精确到微秒级 |
| CPU占用 | 100% | 100%（轮询模式），但时间精确 |
| 抗主频变化 | 差 | 不受主频变化影响 |
| 适用场景 | 粗略延时 | 精确时序控制 |

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| 延时时间不准确 | GPT时钟源配置错误 | 确认GPT时钟源为IPG时钟（66MHz） |
| 延时不工作 | GPT未启动 | 检查`GPT1_CR[EN]`是否置位 |
| 延时过长 | 预分频设置错误 | 检查`GPT1_PR`预分频寄存器配置 |
| 延时精度不足 | 轮询间隔过大 | 使用更高精度的时钟源或减小轮询间隔 |
