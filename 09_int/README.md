# 09_int — 系统中断实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第九个实验——**系统中断配置**的完整代码。
该实验在时钟配置基础上，新增中断系统（GIC，Generic Interrupt Controller）和外部中断（exit）驱动模块，学习ARM Cortex-A7的中断机制。
实验通过按键触发外部中断，在中断服务函数中控制蜂鸣器状态，演示中断处理的基本流程。

## 3. 关键原理简述

### 3.1 ARM Cortex-A7中断体系
I.MX6ULL基于ARM Cortex-A7架构，中断由GIC（Generic Interrupt Controller）统一管理：
- **中断ID**：每个中断源有唯一的中断ID号
- **中断类型**：
  - SPI（Shared Peripheral Interrupt）：共享外设中断，如GPIO中断、定时器中断
  - PPI（Private Peripheral Interrupt）：CPU私有外设中断
  - SGI（Software Generated Interrupt）：软件触发中断
- **中断向量表**：异常向量表，存放各异常和中断的入口地址

### 3.2 GPIO外部中断配置
按键外部中断的配置流程：
1. **GPIO中断使能**：在`GPIO1_IMR`寄存器中使能对应引脚的中断掩码
2. **触发方式配置**：在`GPIO1_ICR`寄存器中配置上升沿/下降沿/双边沿触发
3. **GIC中断使能**：通过GIC的`ICDISER`寄存器使能对应中断ID
4. **优先级配置**：通过`ICDIPR`寄存器设置中断优先级
5. **中断处理函数注册**：将中断服务函数注册到中断向量表中

### 3.3 中断服务函数编写
中断服务函数（ISR）需注意：
- **快速响应**：ISR应尽量简短，避免长时间占用CPU
- **状态清除**：中断退出前需清除GPIO中断状态标志（`GPIO1_ISR`），否则会导致中断持续触发
- **上下文保护**：汇编级需保存/恢复寄存器现场（由中断入口代码自动完成）

## 5. 下载与验证步骤

1. 编译生成`int.bin`
2. 使用`imxdownload`工具烧录至SD卡：
```bash
./imxdownload int.bin /dev/sdX
```
3. SD卡插入开发板，设置SD卡启动模式
4. 上电启动

## 6. 预期运行现象

开发板上电后：
- LED0以约500ms为周期闪烁
- 按下KEY0按键时，触发外部中断，蜂鸣器状态翻转

验证标准：按键按下时蜂鸣器立即响应（非轮询方式），说明中断系统配置成功，外部中断正常工作。

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| 中断不触发 | 中断未使能 | 检查`GPIO1_IMR`和GIC的`ICDISER`是否已正确使能 |
| 中断持续触发 | 中断状态未清除 | 在ISR中清除`GPIO1_ISR`对应bit |
| 程序进入中断后卡死 | 中断向量表配置错误 | 检查中断向量表基地址是否正确写入`VBAR`寄存器 |
| 按键无响应 | 触发方式配置错误 | 检查`GPIO1_ICR`中触发方式配置（上升沿/下降沿）是否与按键硬件匹配 |
| 中断优先级异常 | GIC初始化不完整 | 检查`bsp_int.c`中GIC的`ICDDCR`、`ICCBPR`等寄存器配置 |
