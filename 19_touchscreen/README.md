# 19_touchscreen — 触摸屏驱动实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第十九个实验——**触摸屏驱动**的完整代码。
该实验实现电容式触摸屏的驱动，支持5点触摸检测。
触摸屏驱动基于I2C接口与触摸控制器（如FT5X06系列）通信，获取触摸坐标并在LCD上显示触摸点。

## 3. 关键原理简述

### 3.1 电容式触摸屏原理
电容式触摸屏通过检测人体电容变化实现触摸定位：
- **投射式电容**：在玻璃表面形成电容矩阵
- **触摸检测**：手指触摸时局部电容变化
- **坐标计算**：控制器扫描电容矩阵，计算触摸坐标

### 3.2 FT5X06触摸控制器
FT5X06是电容式触摸控制器，特点：
- **接口**：I2C通信
- **触摸点数**：支持5点触摸
- **坐标输出**：X/Y坐标、触摸ID、状态等
- **中断输出**：触摸发生时产生中断信号

### 3.3 触摸数据处理
1. 检测触摸中断
2. 通过I2C读取触摸坐标数据
3. 解析数据包，获取触摸点坐标
4. 在LCD上绘制触摸点标记
5. 支持多点触摸时循环处理各点数据

## 5. 下载与验证步骤

1. 编译生成`touchscreen.bin`
2. 使用`imxdownload`工具烧录至SD卡：
```bash
./imxdownload touchscreen.bin /dev/sdX
```
3. SD卡插入开发板，设置SD卡启动模式
4. 确保触摸屏已正确连接至开发板
5. 上电启动

## 6. 预期运行现象

开发板上电后：
- LCD屏幕显示触摸测试界面
- 手指触摸屏幕时，显示触摸点坐标
- 支持多点触摸时，显示多个触摸点

验证标准：触摸点坐标准确，跟随手指移动，说明触摸屏驱动正常。

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| 触摸无响应 | I2C通信失败 | 检查触摸屏I2C地址和连接 |
| 触摸坐标偏移 | 校准数据错误 | 重新进行触摸屏校准 |
| 触摸漂移 | 干扰或接地不良 | 检查触摸屏接地和屏蔽 |
| 多点触摸异常 | 控制器不支持或配置错误 | 检查FT5X06配置和固件版本 |
