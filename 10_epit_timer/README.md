# 10_epit_timer — EPIT定时器实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第十个实验——**EPIT（Enhanced Periodic Interrupt Timer）定时器**的完整代码。该实验在中断系统基础上，新增EPIT定时器驱动模块，学习I.MX6ULL增强型周期中断定时器的工作原理与配置方法。EPIT定时器通过硬件计数实现精确延时，避免了软件空循环的低效和不准确，是嵌入式系统中实现定时功能的核心外设。

## 3. 关键原理简述

### 3.1 EPIT定时器架构
I.MX6ULL内部集成两个EPIT定时器（EPIT1和EPIT2），特性包括：
- **32位递减计数器**：从预设值递减至0时产生中断
- **多种时钟源选择**：可选择IPG时钟（66MHz）或外部时钟
- **预分频器**：1~4096分频，灵活配置定时周期
- **比较值模式**：支持比较中断和溢出中断两种模式

### 3.2 EPIT配置流程
1. **时钟使能**：通过CCM使能EPIT模块时钟
2. **设置时钟源**：选择IPG时钟（66MHz）作为计数源
3. **设置预分频**：根据需要的定时周期计算预分频系数
4. **设置比较值**：写入`EPIT_LR`寄存器，定义计数起始值
5. **使能中断**：在GIC中使能EPIT中断
6. **启动定时器**：置位`EPIT_CR[EN]`启动计数

### 3.3 定时周期计算
以EPIT1为例，使用IPG时钟（66MHz）：
```
定时周期 = (预分频系数 + 1) × 比较值 / IPG时钟频率
```
例如：预分频=0（1分频），比较值=33000000，则定时周期 = 33000000/66MHz = 500ms

### 3.4 与软件延时的对比
| 特性 | 软件空循环延时 | EPIT硬件定时 |
|------|-------------|-------------|
| CPU占用 | 100%占用 | 零占用，中断触发 |
| 精度 | 受编译优化和主频影响 | 精确，由硬件时钟决定 |
| 功耗 | 高 | 低（CPU可进入低功耗模式） |
| 适用场景 | 简单演示 | 精确定时、多任务调度 |

## 6. 预期运行现象

开发板上电后：
- LED0以约500ms为周期闪烁（由EPIT定时器中断驱动）
- 按键控制蜂鸣器功能正常工作

验证标准：LED闪烁周期精确稳定，不受CPU主频变化影响，说明EPIT定时器配置成功，硬件定时功能正常。

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| 定时器不触发中断 | EPIT时钟未使能 | 检查CCM中EPIT模块时钟门控是否已打开 |
| 定时周期不准确 | 时钟源选择错误 | 确认`EPIT_CR[CLKSRC]`配置为IPG时钟（01b） |
| 中断频繁触发 | 比较值设置过小 | 增大`EPIT_LR`比较值，或增加预分频系数 |
| 定时器启动后无计数 | 计数器未加载初始值 | 确保在启动定时器前已写入`EPIT_LR`寄存器 |
| 多定时器冲突 | EPIT1和EPIT2中断ID混淆 | 确认使用的中断ID（EPIT1为56，EPIT2为57） |
