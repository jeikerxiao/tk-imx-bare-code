# 17_i2c — I2C总线/AP3216C传感器实验

## 1. 目录功能概述

本目录存放I.MX6ULL裸机开发工程第十七个实验——**I2C总线通信与AP3216C传感器驱动**的完整代码。
该实验实现I.MX6ULL I2C1控制器的初始化，并通过I2C接口驱动AP3216C环境光传感器和接近传感器。
I2C是嵌入式系统中应用最广泛的低速串行通信总线之一，掌握I2C驱动开发是连接各类传感器和外设的基础。

## 3. 关键原理简述

### 3.1 I2C总线原理
I2C（Inter-Integrated Circuit）总线特点：
- **两线制**：SCL（时钟线）和SDA（数据线）
- **主从架构**：一主多从，通过地址寻址
- **速率**：标准模式100kbps，快速模式400kbps
- **时序**：起始条件、数据位、应答位、停止条件

### 3.2 I2C寄存器操作
I.MX6ULL的I2C控制器通过以下寄存器实现通信：
- **I2Cx_IADR**：从设备地址寄存器
- **I2Cx_IFDR**：分频寄存器，设置I2C时钟频率
- **I2Cx_I2CR**：控制寄存器，启停I2C通信
- **I2Cx_I2SR**：状态寄存器，反映通信状态
- **I2Cx_I2DR**：数据寄存器，收发数据

### 3.3 AP3216C传感器
AP3216C是三合一传感器：
- **ALS（环境光传感器）**：测量环境光强度
- **PS（接近传感器）**：检测物体距离
- **IR（红外传感器）**：测量红外强度

通信流程：
1. I2C初始化，配置时钟和引脚
2. 向AP3216C写入配置命令
3. 等待转换完成
4. 读取数据寄存器获取传感器值

1. 编译生成`i2c.bin`
2. 使用`imxdownload`工具烧录至SD卡：
```bash
./imxdownload i2c.bin /dev/sdX
```
3. SD卡插入开发板，设置SD卡启动模式
4. 上电启动

## 6. 预期运行现象

开发板上电后：
- LCD屏幕显示AP3216C传感器数据（ALS、PS、IR值）
- 用手遮挡传感器时，PS值变化
- 改变环境光时，ALS值变化

验证标准：传感器数据随环境变化而变化，说明I2C通信正常。

## 7. 常见问题排查

| 故障现象 | 可能原因 | 解决办法 |
|----------|----------|----------|
| I2C通信失败 | 从设备地址错误 | 核对AP3216C的I2C地址（0x1E） |
| 数据读取全0 | 传感器未初始化 | 检查AP3216C初始化命令序列 |
| I2C总线忙 | 时钟拉伸或从设备无响应 | 检查SDA/SCL上拉电阻是否焊接 |
| 数据跳变异常 | 采样率设置不当 | 调整AP3216C配置寄存器中的采样参数 |
